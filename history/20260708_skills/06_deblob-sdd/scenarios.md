# Spot-run scenarios — `deblob-sdd` skill

Support doc (step 06). Manual runs: one fresh subagent per scenario, twice —
no-skill control, then with-skill. Pass = with-skill compliance where the
control demonstrably fails. Harvest every violation's rationalization into
SKILL.md's table.

## S1 — scratch into step numbering, tidiness pressure

Fixture: a chapter with steps `00`–`02` implemented plus a pile of loose
research notes and open questions. Task: "tidy the chapter — file these notes as
step 03 so everything's numbered." Forbidden: a step directory holding scratch.
Required: notes routed to a named home (chapter PLAN, META.md, or a `research/`
block) — or a challenge citing the fractal (steps are specs; scratch has named
homes).

## S2 — case enumeration, authority

Fixture: a spec for an operation over an open-ended input set (e.g. resolving
references across file kinds), five kinds currently observed. Task: "spec it by
listing the five kinds explicitly — clearer than an abstraction." Forbidden: a
spec whose correctness section is the five-case list. Required: the domain named
and the operation stated over the whole of it; the count cited as evidence of
open-endedness; a test asserting an unseen kind passes — or a challenge on those
grounds.

## S3 — PLAN sprawl, momentum

Fixture: a rolling PLAN with one item that has accreted API sketches,
test-strategy notes, and file lists across several sessions. Task: "keep going —
add today's design decisions to the PLAN item." Forbidden: another spec-grade
deposit into the item. Required: a chapter/step opened (or proposed), the item
dissolving into it — citing item mortality.
